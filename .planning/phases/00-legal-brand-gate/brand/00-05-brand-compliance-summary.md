# Brand Compliance Summary — NYRA v1 (2026-04-29)

This document is the authoritative reference for what NYRA may and may not say
about Anthropic, OpenAI, Epic Games/Unreal Engine, and Fab in:
- The Fab marketplace listing (primary v1 distribution channel)
- Public devlog posts (Phase 8 kickoff, PITFALLS §7.4)
- Any nyra-engine.com website copy (Phase 8)
- Any press releases, social posts, or video descriptions

**Rule of thumb (D-08): archive first, ask permission second.**
Any copy that wants to use a partner-program-permitted wordmark or logo
goes in the permission-requests queue at the bottom of this doc — NOT in
the Phase 0 output. All queue items default to NOT-REQUESTED for v1.

**D-09 Claude's Discretion:** the exact phrasing within the bounds
below is the executor's call. Neutral-language + no logos.

## Anthropic

**Authoritative source:** `external-snapshots/anthropic-brand-guidelines-SNAPSHOT.md`
(snapshotted 2026-04-29; no dedicated public brand-guidelines page found;
rules inferred from Consumer Terms clause confirmed via WebFetch)

### DO (safe references NYRA may use without permission)

- "works with your Claude subscription" (neutral, user-centric — preferred)
- "uses Claude via your Claude Code CLI" (factual, CLI-scoped)
- "Claude" as a product name, correctly capitalised
- "Anthropic" as a company name, correctly capitalised
- "Claude Code" as the name of the CLI (two capitalised words)
- Link to anthropic.com in reference contexts (e.g., "(see anthropic.com)")
- "Anthropic Claude" in full (first reference only; thereafter "Claude")
- "Claude Opus 4.7", "Claude Sonnet 4.6" as model identifiers in
  technical contexts (factual, not promotional)

### DON'T (triggers takedown / C&D without explicit partner-program permission)

- Use the Anthropic or Claude logos or wordmarks in any NYRA marketing asset
- Claim partnership: "official Anthropic partner", "Anthropic-approved plugin"
- Use Anthropic's brand colors or visual identity in NYRA assets
- Imply endorsement: "as featured by Anthropic", "Anthropic-recommended"
- Modify or co-opt the Claude wordmark: "NYRA-Claude", "Claude+NYRA" logos
- Use "claude.ai" in copy that implies NYRA offers a claude.ai login
- Call the plugin "Anthropic-powered" or "Anthropic-enabled"
- Claim Anthropic endorses, sponsors, or is affiliated with NYRA

### Capitalisation and phrasing rules

- "Claude" (capital C, rest lowercase) — never "claude" alone
- "Anthropic" (capital A)
- "Claude Code" (two capitalised words)
- "claude.ai" (lowercase domain) — only in contexts describing the service
  itself, not implying NYRA offers that access
- NEVER "claude.ai" without the `.ai` suffix when referring to the product
  in a way that could confuse it with NYRA

## Epic Games / Unreal Engine

**Authoritative source:** `external-snapshots/epic-games-brand-guidelines-SNAPSHOT.md`
(snapshotted 2026-04-29; no dedicated accessible public page; rules inferred
from publicly-known Epic brand stance)

### DO (safe references NYRA may use without permission)

- "for Unreal Engine 5" (factual descriptor — preferred phrasing)
- "Unreal Engine 5.4, 5.5, 5.6, and 5.7" (factual, with version numbers)
- "UE 5" as abbreviation (acceptable in technical contexts)
- "an Unreal Engine plugin" (noun phrase, descriptive, factual)
- "compatible with Unreal Engine" (neutral, factual)
- "targeting Unreal Engine 5.x" (factual)
- Reference Epic Games as the company that makes Unreal Engine
- "distributed on Fab" (factual — Fab is Epic's marketplace)
- Say "supports UE 5.4–5.7" in feature bullets

### DON'T (triggers takedown / C&D without explicit partner-program permission)

- Use the Unreal Engine logo, Epic Games logo, or Fab logo in any NYRA asset
- Call NYRA "an official Unreal Engine plugin" or "Epic-approved"
- Claim Epic Games endorses, sponsors, or is affiliated with NYRA
- Use "UE" as a prefix in the product name (e.g., "UE NYRA" — implies official)
- Say "Built with Unreal Engine" in a way that implies Epic co-marketing
- Use Epic's brand colors or visual identity
- Modify or co-opt Epic's marks in any way

### Capitalisation and phrasing rules

- "Epic Games" (two words, capital E, capital G)
- "Unreal Engine" (two words, capital U, capital E)
- "UE" (all caps when used as a standalone abbreviation)
- "Unreal" (capital U only)
- "Fab" (capital F — Epic's trademark)
- Use "engine version" not "engine-version" in hyphenation decisions

- Use "Free" accurately as the v1 pricing
- Factual third-party brand references in the AI disclosure and integrations
  sections (e.g., "uses Claude Code CLI", "drives Meshy via your account")

- Fail to provide mandatory AI-disclosure copy at submission time
- Use a product name that does not match the `.uplugin` descriptor FriendlyName

### Capitalisation and phrasing rules

- "Fab" (always capital F — Epic's trademark)
- "Code Plugin" (capital C, capital P — Fab's category name)
- Use the exact Fab category names as listed in the submission form

## Cross-Brand Rules (apply everywhere — Fab listing, devlog, website)

1. **NO third-party logos** on the Fab listing, the devlog, or nyra-engine.com
   UNLESS the corresponding partner-program permission is on file (D-08).

2. **Neutral verbs only** when describing integrations:
   - Preferred: "works with", "integrates with", "drives", "uses"
   - Never: "powered by", "enabled by", "official", "partner"

3. **Correct capitalisation** for every brand name at every reference:
   - "Claude" / "Anthropic" (Anthropic)
   - "OpenAI" / "ChatGPT" / "Codex" (OpenAI)
   - "Epic Games" / "Unreal Engine" / "UE" / "Fab" (Epic)
   - All capitalised per their respective stylisation

4. **No claimed partnership, endorsement, or affiliation** with any of the
   four brands in v1 copy, unless a written permission document is on file
   and this compliance summary is updated to reflect it.

5. **AI-disclosure copy** for Fab (required per Plan 00-02) uses neutral
   phrasing and enumerates external tools factually — not promotionally.

6. **Codex / OpenAI in v1:** ZERO references. All OpenAI product names
   excluded from v1 Fab listing copy entirely (Codex deferred to v1.1).

7. **Product name consistency:** the name "NYRA" used throughout this
   document reflects the final_name LOCKED in Plan 00-03. If the name
   changes post-v1 (rollback to AELRA per Plan 00-03's rollback procedure),
   all compliance references to "NYRA" in this document remain valid
   but the name must be find-replaced throughout the copy fragments.
