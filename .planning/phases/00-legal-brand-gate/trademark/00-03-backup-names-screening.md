# NYRA Backup-Names Screening — 5 Candidates (2026-04-24)

## Why this file exists despite NYRA being the primary

Per Task 2 conditional-execute rule: aggregate verdict on NYRA is
**MEDIUM-RISK** (per `00-03-nyra-screening-dossier.md`). Medium-risk
triggers **precautionary** backup screening — candidates are warm-
standby in case a cease-and-desist arrives post-launch. NYRA remains
primary; these five candidates are the rollback list.

Rollback trigger conditions:

1. Founder verbatim-upgrade of Task-1 raw dumps reveals a live Class-9
   NYRA software mark anywhere globally (would flip aggregate verdict
   to BLOCKED retroactively).
2. Post-launch cease-and-desist arrives from New York Racing Association
   (LOW probability; unique-classes analysis favors NYRA-the-plugin, but
   an aggressive C&D is technically possible given NYRA's acronym-
   identical match).
3. A cosmetics/apparel NYRA mark holder files a defensive Class-9
   extension post-NYRA-launch (LOW probability; defensive extensions
   typically stay within apparel/accessory classes for fashion houses).
4. Any post-launch collision discovery during counsel review.

## Candidate selection criteria (per CONTEXT.md §specifics)

1. **Short** — 4–6 letters preferred for typability + visual brand.
2. **Greek/Latin/myth-adjacent OR thematically linked to "agent that
   reaches outside UE to bring things back"** — matches NYRA's
   positioning (the original NYRA name was selected for Greek-
   mythological resonance).
3. **Low fashion/cosmetics prior-art density** — explicitly avoids the
   cluster where NYRA has MEDIUM-RISK exposure (Classes 3, 14, 25, 43).
4. **Phonetically distinct from NYRA** — rules out names that sound
   like NYRA (NARA, NIRA, NORA, NURA are TOO close). A backup must be
   clearly different from the primary so a post-launch rollback
   doesn't just land in the same collision cluster.
5. **Domain + GitHub + social available** (or non-premium if domain is
   taken).

## Selection Rationale

**Selected: AELRA.**

AELRA is selected as the warm-standby backup for NYRA because it wins
on all four dimensions simultaneously: trademark cleanliness (coined
from Greek `aer` but not a known given-name/brand, so no cosmetics/
fashion prior-art density), domain/GitHub availability (non-premium
pricing expected because the coined word has no domainer pressure),
phonetic distance from NYRA (different vowel cluster — AYL-ra vs.
NEE-ra, avoiding the NARA/NURA/NIRA phonetic-bucket pile-up that would
defeat the purpose of a rollback), and aesthetic fit (air/breath/reach
metaphor maps cleanly onto NYRA's "agent that reaches outside UE to
complete the loop" value proposition).

The runner-up VYRELL is a strong alternative if AELRA's verbatim
screening reveals an unexpected Class-9 collision at founder-follow-up
time, but coined-word branding carries higher adoption friction than
mythologically-rooted names (Greek-myth resonance reads as intentional;
coined-sci-fi reads as placeholder).

CAELUM loses on domain-availability — high-value Latin-word domains are
extensively registered. PYRRA loses on the GitHub `pyrra-dev/pyrra` OSS
collision (a fresh NYRA plugin in the open-source UE ecosystem would
perpetually compete with an existing SRE SLO tool for search ranking)
plus the Pyrrha Jewelry alt-spelling precedent. LIVIA is rejected
outright on given-name density.
