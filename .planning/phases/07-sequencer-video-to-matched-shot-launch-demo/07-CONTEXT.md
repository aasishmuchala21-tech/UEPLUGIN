# Phase 7 — Context: Sequencer + Video-to-Matched-Shot (LAUNCH DEMO)

**Phase goal (from ROADMAP):** NYRA delivers DEMO-02 end-to-end — user pastes a YouTube link or attaches a ≤10s mp4, NYRA extracts keyframes, infers composition + lighting + approximate geometry, and assembles a matching single-shot single-location UE scene with one CineCamera, key+fill lighting, sky+fog, and Sequencer-driven camera. This is the launch demo that no competitor ships.

**User decisions (from /gsd-discuss-phase):** NONE YET — this CONTEXT.md generated at planning time.

**Phase 7 success criteria (from ROADMAP):**
1. DEMO-02 ships end-to-end — no competitor has video→matched-shot as of April 2026
2. SCENE-02: create ULevelSequence, add CineCamera actors, set keyframes on transform/intensity/color, shot blocking from NL ("slow push-in, then cut wide")
3. Video Reference Analyzer: ffmpeg scene-cut detection + semantic lighting extraction + camera-move taxonomy (static/pan/tilt/dolly/truck, user-confirmable)
4. Ephemeral video processing: yt-dlp download → extract ≤16 keyframes → Claude vision → /tmp cleanup
5. First-install cold-start test: 3 consecutive random clips pass

**Requirements from REQUIREMENTS.md:**
- **SCENE-02:** Sequencer automation — ULevelSequence creation, CineCamera spawning, keyframe authoring on camera/light/PPV params, NL shot blocking ("slow push-in, then cut wide")
- **DEMO-02:** LAUNCH DEMO — YouTube link / mp4 ≤10s → keyframe extraction → composition/lighting/geometry inference → single-shot UE scene + Sequencer camera

**Dependencies:**
- Phase 6 (scene assembly + lighting): SCENE-01 lighting tools must be complete before SCENE-02 starts
- Phase 5 (external integrations): yt-dlp + ffmpeg bundled from Phase 5
- Phase 4 (Tool Catalog): Actor spawn/transform tools, asset search tools
- Phase 3 (RAG): Claude Opus backend for vision analysis

**Key observations:**
1. SCENE-02 must build ON TOP of Phase 6's SCENE-01 lighting authoring tools — reuse `LightingParams` dataclass, extend with Sequencer-specific fields (camera FOV, focus distance, aperture, camera motion curve)
2. DEMO-02 is the LAUNCH DEMO — every task must be production-quality, no stubs
3. Wave 0 (07-00) is the foundation: ULevelSequence creation + CineCamera binding + keyframe authoring. This must work reliably before Waves 1-2 build the inference pipeline on top.
4. Video Reference Analyzer (Wave 1) uses Claude Opus 4.7 vision — send ≤16 keyframes as images, receive structured JSON describing: composition, lighting, camera movement, geometry categories
5. Ephemeral processing is a hard requirement: yt-dlp downloads to /tmp, NYRA extracts keyframes, sends to Claude, deletes video file after processing. Full keyframe set stays in /tmp for audit but video is deleted within 5 minutes of paste.
6. Camera-move taxonomy (Wave 2) needs user confirmation before committing — show "I detected a slow push-in (dolly), camera at 35mm equivalent, do you want to proceed?" before Sequencer builds the camera motion.
7. Phase 5 bundled yt-dlp + ffmpeg in Binaries/ThirdParty/ — reference Phase 5 patterns for subprocess spawning and path resolution.
8. Cold-start test (Wave 4) must pass on 3 consecutive random clips — this is the release gate per ROADMAP SC#5.
