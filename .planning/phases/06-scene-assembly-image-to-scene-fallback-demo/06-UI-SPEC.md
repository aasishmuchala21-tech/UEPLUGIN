# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Scene Assembly + Image-to-Scene (Fallback Launch Demo). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (UE Slate/UMG) |
| Preset | not applicable |
| Component library | UE Slate (FSlateAttribute pattern) + UMG EditorUtilityWidget |
| Icon library | UE Slate's `FSlateGameIcon` / `FSlateIcon` — use `FStyleManager::GetIcon` for standard editor icons |
| Font | Manrope (headlines, labels) + Inter (body) — loaded via `FSlateFontInfo` with engine font proxy fallback to `Roboto` if fonts are not bundled |

**Design direction source:** `feedback_design_direction.md` (NYRADNA Director cinematic luxury tokens — glassmorphism, lavender `#c6bfff`, Manrope+Inter, ultra-wide tracking).

**No React/web stack.** This is a native UE plugin. All UI lives in Slate panels and UMG `EditorUtilityWidget` Blueprints. Tokens are applied via `FSlateAttribute` / `FLinearColor` / `FMargin` constants in C++ and `UMG` widget style overrides.

## Spacing Scale

Declared values (multiples of 4, aligned to UE's standard 8-unit grid):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon internal gaps, single-element padding |
| sm | 8px | Compact element spacing, chip internal padding |
| md | 16px | Default element spacing, standard panel padding |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Major layout gaps between major blocks |
| 2xl | 48px | Panel-to-panel spacing, dialog margins |
| 3xl | 64px | Dock panel min-height, full-page breathing room |

**Exceptions:** None for this phase. Icon-only touch targets default to 32px minimum (UE standard), not 44px (mobile constraint not applicable on desktop).

## Typography

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (Regular) | 1.5 (SFallbackWidgetStyle default) | Primary readable text in chat, log, asset list |
| Label | 11px | 400 (Regular) | 1.0 | Uppercase, `letter-spacing: 0.15em` — section headers, status tags. Differentiated from Body via letter-spacing (0.15em) not weight. |
| Heading | 18px | 600 (Semibold) | 1.2 | Panel titles, section labels like "Reference Image", "Lighting Setup" |
| Display | 24px | 600 (Semibold) | 1.2 | Modal headings, "Scene Assembled" success heading. Visual hierarchy from 24px size — weight not needed for distinction. |

**Implementation note:** UE Slate `FSlateFontInfo` maps to `Roboto` by default. For Manrope/Inter, bundle `.ttf` files in `NyraEditor/Resources/Fonts/` and create `FSlateFontInfo` with `TypefaceFontName = "Manrope"` / `"Inter"`. Fallback to `Roboto` if fonts not found — do not block on font availability.

**Letter-spacing** (NYRADNA signature): Label role uses 0.15em via `FNumberFormattingOptions::SetSignedZeroSpace(true)` or custom `FSlateAttribute` where needed. Heading uses 0.05em. Display uses 0em (tightening not needed since 24px provides sufficient hierarchy).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#050507` (near-black) | Chat panel background, dock panel base — matches UE dark editor theme |
| Secondary (30%) | `#0D0D14` (dark slate) | Card backgrounds, sidebar panels, asset pill chips |
| Accent (10%) | `#C6BFFF` (soft lavender) | See reserved-for list below |
| Accent container | `#8C80FF` | Hover states, selected asset chips, progress fill |
| Destructive | `#FF4D6A` | Destructive actions only (delete scene, cancel in-progress assembly) |

**Accent reserved for:**
1. Primary button backgrounds (`SNyraPrimaryButton`)
2. Active nav item right-border (3px solid, not left — per NYRADNA brand direction)
3. Focus rings on all interactive Slate widgets (1px `SlateColor` with `#C6BFFF` at 60% opacity)
4. Progress bar fill during scene assembly
5. Link text in chat messages and inline asset names
6. "Assembling..." status pill background
7. Reference image drop zone dashed border when dragged-over

**Secondary surface** `#0D0D14` is used for:
- Glass cards: `FSlateBrush` with `TintColorAndOpacity = FLinearColor(1,1,1,0.04)` + `Margin = FMargin(0,0,1,1)` top-right border in `#C6BFFF` at 25% opacity (glass overlay effect on modal dialogs)
- Chat message bubbles (user vs. agent differentiation via alpha)
- Asset chip backgrounds (selected vs. unselected via alpha toggle)

**Do NOT use accent for:**
- Large background fills (would overwhelm the near-black editor aesthetic)
- Body text color (only link text uses accent)

## Interaction Patterns

### Image Drop Zone (Primary Entry Point — DEMO-01)
1. User drags image over the panel OR pastes from clipboard (Ctrl+V)
2. Drop zone transitions: idle (subtle dashed border) → drag-over (lavender border pulses, glass overlay brightens) → loading (spinner + "Analyzing..." text)
3. On analysis complete: thumbnail appears in Reference Image strip, chat auto-scrolls with agent message "I see a [scene type]. Assembling now..."
4. On error: drop zone shows error state with red border, error copy below

### Scene Assembly Progress (DEMO-01 Execution)
1. Progress bar appears below the reference thumbnail
2. Steps shown as labeled segments: "Placing Actors (3/12)" → "Applying Materials (1/3)" → "Setting Up Lighting" → "Finalizing"
3. Each step fires a streaming chat update from the agent
4. On completion: success state (green checkmark icon in status pill), log drawer auto-expands briefly to show summary
5. On failure: error state (red status pill), partial progress evidence shown, log drawer stays open

### Lighting Authoring Panel (SCENE-01)
1. Opened from chat command or "Match Image Mood" button
2. Shows reference thumbnail + detected mood tags ("warm", "high-contrast", "soft fill")
3. Lighting preset cards: horizontal scroll, user can pick a preset OR click "Match from Image" to apply detected mood
4. Preview updates in UE viewport in real-time as user hovers over presets (dry-run, no commit until "Apply")
5. "Apply Lighting" button fires the SCENE-01 tool call

### Safe-Mode Preview (CHAT-04 inherited from Phase 2)
1. Before any scene mutation, agent outputs a preview card in the chat
2. Preview card shows: actors to place (with names/paths), materials to apply, lighting to configure
3. User can Approve / Edit / Reject via three buttons below the preview
4. On Approve: execution proceeds; on Reject: agent re-plans
